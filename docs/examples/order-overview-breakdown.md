# How the page comes together

Instead of designing the page first, start with the pieces.

Each piece has a job.

Put those pieces together, and you get the full experience.

---

## The core pieces

- Notifications → what needs my attention  
- Order summary → what I bought and what I paid  
- Delivery status → where my order is  
- Recent activity → what just changed  
- Help → what to do if something's wrong  

---

## Why this works

It keeps things simple.

You're not guessing what goes on the page.  
You're building it from real customer needs.

---

## What this helps with

- clearer decisions about what belongs on the page  
- better alignment between content, design, and development  
- easier to scale across different order types and states  

---

## The shift

Instead of:

👉 "What should this page look like?"

You start with:

👉 "What does the customer need right now?"

That changes everything.
